## 10. NOTES FOR APPLE REVIEW TEAM

### 10.1 App Overview
Camlica360 is a B2B (Business-to-Business) HR management application. It's not a consumer app but rather an enterprise application used by employees of companies that use the Camlica CRM system.

The app is currently in PRIVATE BETA and is being submitted for TestFlight review before full App Store release.

### 10.2 Important Notes

1. **Authentication Required:**
   - All app features require login through the Camlica CRM backend
   - Users must have valid credentials in the CRM system
   - OTP verification is required for security

2. **Backend Dependency:**
   - The app depends on an external API (Camlica CRM) for all functionality
   - Without valid backend credentials, the app cannot be fully tested
   - The provided demo account is for testing purposes only

3. **Internet Connection:**
   - The app requires an active internet connection to function
   - It cannot work offline (as all data comes from the backend)

4. **Data Privacy:**
   - The app handles employee personal data (names, leave records, approval status)
   - All data is encrypted in transit (HTTPS)
   - Sensitive data (tokens) are stored securely in Keychain

5. **Supported Languages:**
   - English (en)
   - Turkish (tr)
   - Language detection is based on device settings

6. **Permissions Used:**
   - Local Network Access (for LAN connectivity - can be disabled if not needed)
   - No camera, microphone, location, or contact permissions required

### 10.3 App Content Rating

### 10.4 Testing Instructions

**Step 1: Launch & Login**
1. Open the app
2. You'll see the splash screen followed by the login screen
3. Enter demo credentials (Company Code: DEMO, ID Number: 12345, Password: Demo@123456)
4. Click "Login"

**Step 2: OTP Verification**
1. After successful login, you'll be prompted for OTP verification
2. The OTP should be delivered via configured method (email/SMS)
3. Enter the OTP and click verify
4. Or provide us with a test OTP if your backend supports static test codes

**Step 3: Explore Main Features**
1. Home Tab: View user welcome and profile card
2. İzinler Tab: View leave dashboard with charts, approvals, and leave requests

**Step 4: Test Leave Request Creation** (if demo account has no active requests)
